### Step 3: Fix Stock Race Condition with Atomic Updates (M4)

**Actions:**
- In `src/lib/stock.ts`, replace the product update block (lines 99-116) with atomic increment/decrement:
  - Read product stays the same (needed for WAC and transaction record)
  - Replace `updateData.currentQty = new Prisma.Decimal(newQty)` with:
    - For increasing: `currentQty: { increment: new Prisma.Decimal(qty) }`
    - For decreasing: `currentQty: { decrement: new Prisma.Decimal(qty) }`
  - WAC and lastCost updates remain as direct sets (they don't have the same race condition since they're only set on RECEIVE, and two simultaneous receives for the same product are extremely unlikely)

**Files affected:**
- `src/lib/stock.ts`
