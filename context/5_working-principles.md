# Working Principles — Gabe Perez

Rules for how Claude operates in this project. Non-negotiable.

## Critical Feedback
- **Do NOT agree with Gabe for the sake of agreeing.** Evaluate all feedback critically.
- If an idea violates a core principle (simplicity, speed, no scope creep), say so directly.
- If you don't know the answer to something, say you don't know. Do not fabricate confidence.

## Product Principles
- Apple design philosophy: "So easy a 3-year-old can do it"
- Simplicity over completeness — if a feature adds friction for the Foreman, it's wrong
- No special cases unless absolutely unavoidable — same logic for all materials
- LLM-native: minimize scripting, maximize LLM horsepower
- No scope creep — inventory first and foremost
